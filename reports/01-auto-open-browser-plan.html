Auto-Open Browser on Server Start

Technical Implementation Plan

January 6, 2026

Overview

Add Vite-like browser auto-open functionality when the development or CLI server starts. This enhances developer experience by automatically launching the browser to the correct URL, reducing friction in the development workflow.

Goals

  • 1 Automatically open the default browser when servers start
  • 2 Support both development server (ui/index.ts) and CLI production server (cli/server.ts)
  • 3 Provide a --no-open flag for CLI to disable auto-open behavior
  • 4 Cross-platform support (macOS, Windows, Linux)

Files to Create

cli/open-browser.ts
New File

Cross-platform utility function to open URLs in the default browser. Uses native OS commands: open (macOS), start (Windows), xdg-open (Linux). Uses Bun.spawn() with stdout/stderr ignored. 

export function openBrowser(url: string): void {
  const command = process.platform === "darwin"
    ? "open"
    : process.platform === "win32"
      ? "start"
      : "xdg-open";

  Bun.spawn([command, url], {
    stdout: "ignore",
    stderr: "ignore",
  });
}

Usage Examples 

# Development - auto-opens browser
bun dev

# CLI - auto-opens browser (default)
iaee run experience.json

# CLI - disable auto-open
iaee run experience.json --no-open

Design Decisions

1

Shared utility in cli/ folder

Since there's no existing lib/ folder and this is CLI-focused, keeping it in cli/ is pragmatic

2

Default to open

Following Vite's convention, auto-open is enabled by default

3

--no-open flag

Negated flag pattern (vs --open) because opening is the default behavior

4

No server readiness check

Bun.serve() is synchronous, so the server is ready immediately

Implementation Summary

1 new file, 3 modified files. Minimal footprint with maximum developer experience improvement.
