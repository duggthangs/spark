# IAEE Output Reference: Markdown Summary Guide

This document defines how user interactions are summarized into Markdown. Agents consuming the output of IAEE should use this as a reference for parsing user feedback.

## 1. Summary Structure
Every report follows a consistent hierarchy:
1. **Title** (H1)
2. **Global Description** (Optional)
3. **Horizontal Rule** (`---`)
4. **Section Summaries** (H3 for each section)
5. **Final Decision** (H3 + Emoji status)

---

## 2. Section Mappings

| Section Type | Resulting Markdown | Logic / Meaning |
| :--- | :--- | :--- |
| **`info`** | `> Content` | Provides context for the subsequent decisions. |
| **`choice`** | `**Selected:** label1, label2` | Comma-separated list of user-selected option labels. |
| **`rank`** | `1. Item A` <br> `2. Item B` | Numbered list reflecting the user's priority order. |
| **`text-review`**| `> Original Content` <br> `**Feedback:** User comment` | User's reaction to a specific text proposal. |
| **`kanban`** | `### Column Name` <br> `- Item` | Items grouped under their respective column headers. |
| **`image-choice`**| `**Selected:** Label (ID)` | The specific image variant the user approved. |
| **`api-builder`** | ` ```http ` <br> `POST /path` | A code block containing the refined API contract. |
| **`data-mapper`** | `Source → Target` | A table or list showing specific field mappings. |
| **`live-component`**| ` ```tsx ` <br> `<div...>` | The **final JSX code** generated by the user's styling. |
| **`numeric-inputs`**| `- Item: Value` | Captured numeric values (e.g. budget allocations). |
| **`card-deck`** | `- Name: { Field: Value }` | Details filled out within interactive persona/info cards. |
| **`decision`** | `✅ Status: Approved` <br> `❌ Status: Rejected` | The final approval gate status + rationale. |

---

## 3. Empty States
If a user skips a section or leaves it at default, IAEE uses standardized "empty" indicators:
- `*No selection*` (Choice, ImageChoice)
- `*No items*` (Rank, Kanban)
- `*No code generated*` (Live Component, API Builder)
- `*No feedback provided*` (Text Review)

## 4. Example Report

```markdown
# Database Migration Plan

Plan to move from SQLite to Postgres.

---

### Stack Selection
**Selected:** Postgres, Prisma

### Migration Steps
1. Create Schema
2. Data Export
3. Verification

### API Changes
```http
GET /api/v1/health
```

### Final Decision
✅ **Status:** Approved
**Rationale:** Looks solid, proceed with the weekend window.
```
